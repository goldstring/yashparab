Car -Purchase Decision Classification

Project Resources

  • Kaggle Notebook : Click Here

In this project, I developed a machine learning model to predict loan approval based on an applicant's financial and demographic data. The aim was to create a classification model to help financial institutions improve their decision-making processes for loan applications.

Problem Statement:

The objective was to accurately predict whether a loan application would be approved or rejected, using features such as income, credit history, loan amount, and applicant details. The model helps in risk assessment and streamlining the loan approval process for banks and financial organizations.

Dataset Overview:

  • Loan_ID:
    Unique identifier for each loan application.
  • Gender:
    Gender of the applicant.
  • Married:
    Applicant's marital status.
  • Dependents:
    Number of dependents.
  • Education:
    Education level (Graduate/Non-Graduate).
  • Self_Employed:
    Whether the applicant is self-employed or not.
  • ApplicantIncome & CoapplicantIncome:
    Applicant and co-applicant income.
  • LoanAmount & Loan_Amount_Term:
    Requested loan amount and loan repayment term.
  • Credit_History:
    History of credit score (whether the applicant has a good credit history or not).
  • Loan_Status:
    Target variable (1 for approved, 0 for rejected).

Steps Involved:

Exploratory Data Analysis (EDA):
  • Explored data distributions and relationships between loan approval and features like income, loan amount, and credit history.
  • Addressed missing values in important columns like LoanAmount and Credit_History using median imputation.
  • Visualized relationships with survival rates using heatmaps, bar plots, and boxplots.
Data Preprocessing:
  • Categorical variables (Gender, Education, Married) were converted to numeric codes using Label Encoding.
  • Scaled numerical variables like ApplicantIncome and LoanAmount using StandardScaler.
  • Managed class imbalance using SMOTE to ensure even representation of approved and rejected loans.
Feature Engineering:
  • Combined features like ApplicantIncome and CoapplicantIncome to create a Total_Income variable.
  • Binned continuous variables like LoanAmount to reduce variability and enhance model performance.
Model Building:
  • Initially tested models like Logistic Regression, Decision Trees, and Random Forest.
  • Final model: GradientBoostingClassifier which outperformed other models with an accuracy of 89% on the test set.
  • Gradient Boosting was chosen due to its ability to handle non-linear relationships and optimize for better predictive power.
Hyperparameter Tuning:
  • Used GridSearchCV to tune hyperparameters like the learning rate, the number of estimators, and the maximum depth of the trees for the Gradient Boosting model.
  • The fine-tuned GradientBoostingClassifier resulted in a significant performance boost.
Model Evaluation:
  • Evaluated the model using a Confusion Matrix and ROC curve to analyze precision, recall, and overall performance.
  • Achieved a final accuracy of 89% with the GradientBoostingClassifier, providing a strong, reliable model for loan approval predictions.

Conclusion:

This project gave me practical experience in applying machine learning techniques for classification tasks, especially using Gradient Boosting to achieve high accuracy. It strengthened my understanding of how to handle real-world datasets, clean data, perform feature engineering, and optimize models for the best possible results.

Key technologies used: Python, Pandas, Numpy, Scikit-learn, Matplotlib, Seaborn, GradientBoostingClassifier.
